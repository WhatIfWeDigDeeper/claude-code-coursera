# How to Add and Edit Expenses

## What is the Expense Form?

The Expense Form is where you add your daily expenses to track your spending. It's a simple, easy-to-use form that helps you quickly record what you spent, when you spent it, and what it was for.

## Key Benefits

- ‚úÖ Quick entry - Add an expense in under 30 seconds
- ‚úÖ Smart defaults - Today's date is pre-filled
- ‚úÖ Validation - Prevents common mistakes like negative amounts
- ‚úÖ Edit anytime - Easily fix mistakes or update expenses
- ‚úÖ Works in dark mode - Comfortable viewing day or night

## Getting Started

### Prerequisites

- None! The form is ready to use immediately
- Works on all modern browsers (Chrome, Firefox, Safari, Edge)
- Responsive design works on desktop, tablet, and mobile

1. Open the Expense Tracker application

## How to Add a New Expense

### Step 1: Select the Date

The date field shows today's date by default.

[SCREENSHOT PLACEHOLDER: Date field]
![Date Selection](../screenshots/expense-form-date.png)
*Screenshot showing the date picker*

**What to do**:
1. Click on the date field
2. Either type the date in YYYY-MM-DD format
3. Or use the calendar picker to select a date

**Tips**:
- You can add expenses from the past (like yesterday's coffee)
- You can add future expenses (like an upcoming subscription)
- The date defaults to today to save you time

### Step 2: Enter the Amount

Enter how much you spent (without the dollar sign).

[SCREENSHOT PLACEHOLDER: Amount field]
![Amount Entry](../screenshots/expense-form-amount.png)
*Screenshot showing amount input*

**What to do**:
1. Click in the "Amount" field
2. Type the number (e.g., `45.99` or `12.50`)
3. Don't include the $ symbol - just the number

**Tips**:
- Use decimals for cents (12.50, not 12.5)
- The app prevents you from entering $0 or negative numbers
- You can use amounts like 1234.56 for larger purchases

**Common mistake to avoid**:
- ‚ùå Don't type "$45.99" - just type "45.99"

### Step 3: Choose a Category

Select the category that best matches your expense.

[SCREENSHOT PLACEHOLDER: Category dropdown]
![Category Selection](../screenshots/expense-form-category.png)
*Screenshot showing the category dropdown menu*

**What to do**:
1. Click on the "Category" dropdown
2. Select from these options:
   - **Food** - Groceries, restaurants, coffee
   - **Transport** - Gas, bus fare, Uber, parking
   - **Entertainment** - Movies, games, hobbies
   - **Utilities** - Electric, water, internet bills
   - **Healthcare** - Doctor visits, medicine, insurance
   - **Other** - Anything that doesn't fit above

**Tips**:
- When in doubt, use "Other"
- The default is "Food" since it's commonly used
- Categories help you see spending patterns in the Dashboard

### Step 4: Add a Description

Write a short note about what the expense was for.

[SCREENSHOT PLACEHOLDER: Description field]
![Description Entry](../screenshots/expense-form-description.png)
*Screenshot showing description field*

**What to do**:
1. Click in the "Description" field
2. Type a brief note (e.g., "Lunch at Subway" or "Gas for car")
3. Keep it short but descriptive enough to remember

**Tips**:
- Be specific enough to remember later ("Coffee" vs "Starbucks latte")
- You can use emojis if you like (üçï Pizza night)
- This field is required - can't be left empty

**Common mistake to avoid**:
- ‚ùå Don't just type spaces - write an actual description

### Step 5: Save the Expense

Once all fields are filled, click the save button.

[SCREENSHOT PLACEHOLDER: Save button]
![Save Button](../screenshots/expense-form-save.png)
*Screenshot showing the "Add Expense" button*

**What to do**:
1. Click the blue "Add Expense" button
2. Or press Enter on your keyboard

**What happens next**:
- The expense is saved instantly
- The form clears and is ready for another entry
- You'll see the expense appear in the "Expense List" tab
- The dashboard updates with your new expense

**Tips**:
- If there's an error, you'll see a red message below the field
- Fix the error and click "Add Expense" again
- The form won't save until all fields are valid

## Common Use Cases

2. Click "Add Expense" tab (the date is already today)
3. Type `12.50` in Amount

## Tips and Best Practices

### Do's ‚úÖ

- **Add expenses regularly** - Try to add them daily so you don't forget
- **Be descriptive** - "Target groceries" is better than just "groceries"
- **Use the right category** - This helps with budget tracking
- **Check your entries** - Review the expense list to catch mistakes early
- **Edit mistakes immediately** - Don't wait to fix errors

### Don'ts ‚ùå

- **Don't forget the cents** - Use 12.50 instead of 12 for accuracy
- **Don't leave fields empty** - All fields except date are required
- **Don't use the wrong category** - Your analytics will be off
- **Don't enter negative amounts** - The app won't let you anyway
- **Don't type the $ symbol** - Just enter the number

### Pro Tips üí°

- **Keyboard shortcut**: Press Enter to submit the form quickly
- **Tab key**: Use Tab to move between fields without the mouse
- **Mobile friendly**: Add expenses on your phone right after purchasing
- **Batch entry**: Set aside 5 minutes to add all your expenses at once
- **Check the dashboard**: After adding expenses, check the Dashboard tab to see your spending patterns

## Troubleshooting

### Problem: "Amount must be greater than 0" error

**Symptoms**: Red error message appears below the Amount field

**Solution**:
1. Make sure you entered a number
2. Check that the number is greater than 0
3. Remove any dollar signs ($) or commas (,)
4. Use format like: 45.99 or 12.50

**Why it happens**: The app requires positive amounts to prevent data entry errors

---

### Problem: "Description is required" error

**Symptoms**: Red error message appears below the Description field

**Solution**:
1. Type something in the description field
2. Don't just type spaces - write actual text
3. Example: "Coffee at Starbucks"

**Why it happens**: Expenses need descriptions to be meaningful and searchable

---

### Problem: Form won't save / Nothing happens when I click Add

**Symptoms**: Button click doesn't do anything

**Solution**:
1. Check all fields for red error messages
2. Make sure date field has a value
3. Verify amount is a number greater than 0
4. Ensure description isn't empty
5. Fix any errors and try again

**Why it happens**: The form validates all fields before saving

---

### Problem: Can't exit edit mode

**Symptoms**: Form is stuck showing an expense's data

**Solution**:
1. Click the "Cancel" button below the "Update Expense" button
2. This returns the form to normal "Add" mode
3. The form clears and is ready for new entries

**Why it happens**: Edit mode stays active until you save or cancel

---

### Problem: Decimal amounts aren't working

**Symptoms**: Can't enter cents, only whole dollars

**Solution**:
1. Use a period (.) not a comma (,) for decimals
2. Format: 12.50 ‚úÖ not 12,50 ‚ùå
3. This is standard US number formatting

**Why it happens**: The app expects decimal points, not commas

## Frequently Asked Questions

### Can I add expenses from last month?

Yes! Click the date field and select any date in the past. The app doesn't restrict how far back you can go.

### How do I delete an expense instead of editing it?

Go to the Expense List tab and click the red "Delete" button next to the expense. (The Edit button opens the form, the Delete button removes it.)

### Can I add the same expense twice?

Yes, you can add multiple expenses with the same details. Each expense is saved separately with a unique ID.

### What happens if I close the browser?

Your expenses are saved automatically to your browser's storage. They'll be there when you come back.

### Can I add expenses without an internet connection?

Yes! The app works completely offline. Everything is saved in your browser.

### Why does the form clear after I save?

This is by design so you can quickly add another expense without manually clearing the fields.

### Can I change the categories?

Not through the UI currently. The categories are: Food, Transport, Entertainment, Utilities, Healthcare, and Other.

### Is there a limit to how many expenses I can add?

No hard limit, but browser storage has limits (usually several thousand expenses is fine).
